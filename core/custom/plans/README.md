# Plans

This directory stores execution plans and operational memory for non-trivial work.

## Directories

- `core/custom/plans/active/` - current work in progress
- `core/custom/plans/completed/` - finished plans worth preserving
- `core/custom/plans/backlog/` - deferred ideas and planned investigations

## When To Create A Plan

- the task spans multiple sessions
- the task touches several subsystems
- the task needs a decision log
- the task needs explicit verification steps

## Suggested Plan Structure

- Title
- Status
- Date
- Goal
- Context
- Constraints
- Steps
- Risks
- Decision Log
- Verification

## Naming

Use a date prefixed filename when possible, for example:

- `2026-03-17-template-context-audit.md`
- `2026-03-17-live-project-verification.md`
