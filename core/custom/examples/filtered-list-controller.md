# Example Filtered List Controller

> Important: this is a reusable example pattern, not a drop in truth for every project.
> Before applying it, verify the real implementation on the live project.

## Why Keep This Example

- Many Evo projects have list pages with `DocLister`, pagination, and GET based filters.
- A stable example helps keep cache logic and filter handling consistent.

## Example Shape

```php
<?php

namespace EvolutionCMS\Main\Controllers;

use EvolutionCMS\Main\Helper;

class NewsListController extends BaseController
{
    protected function setPageData(): void
    {
        $this->data['news'] = $this->getNews();
    }

    protected function getNews(): array
    {
        $key = 'News_' . ($_GET['page'] ?? '1');
        $params = [
            'parents' => 5,
            'display' => 6,
            'paginate' => 'pages',
            'config' => 'paginate:custom',
            'tvList' => Helper::newsTvList(),
        ];

        $filters = [];

        if (!empty($_GET['types'])) {
            $types = implode(',', array_map('intval', explode(',', $_GET['types'])));
            $this->data['filter']['types'] = array_map('intval', explode(',', $_GET['types']));
            $filters[] = 'tv:sportTag:in:' . $types;
        }

        if (!empty($filters)) {
            $key = false;
            $params['filters'] = 'AND(' . implode(';', $filters) . ')';
        }

        $rows = Helper::DocLister($key, $params);
        $this->data['pages'] = $this->getPages($key);

        return $rows;
    }
}
```

## Working Rule

- Keep request parsing in the page controller when filters are page specific.
- Disable or vary cache keys when request filters change the query.
- Pass normalized filter state into the view so the UI can reflect active filters.

## Verify On Live Project

- whether request filters come from `$_GET`, routes, or request wrappers
- whether pagination placeholders are generated by the same query
- whether filter values target TVs, fields, or custom DocLister controllers
- whether caching should be disabled, varied, or time limited for filtered output
